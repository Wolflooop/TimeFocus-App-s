# Experiencia de Usuario (UX/UI) — TimeFocus

En esta sección se encuentra documentado el proceso de diseño centrado en el usuario para la plataforma **TimeFocus**. El objetivo principal es garantizar una interfaz intuitiva, accesible y eficiente para estudiantes y cualquier usuario que desee gestionar su tiempo de forma efectiva.

## Contenido de esta carpeta

La carpeta `docs/uiux/` está organizada en las 4 capas del proceso de diseño en Figma:

### 1. Sketches
Bocetos iniciales a mano alzada o digitales que representan las primeras ideas de la interfaz. En esta etapa se exploran estructuras sin preocuparse por colores ni estilos.

### 2. Wireframes
Estructura visual de las pantallas principales en baja y alta fidelidad. Define la disposición de elementos como la barra de navegación, el calendario de tareas, el panel de sesiones y el dashboard de estadísticas.

### 3. Mockups
Diseño de alta fidelidad con identidad visual aplicada — paleta de colores, tipografías, iconografía y componentes finales. Representa cómo se verá la aplicación en producción.

### 4. Prototipo Navegacional
Flujo de navegación interactivo que conecta todas las pantallas. Permite simular la experiencia real del usuario al registrarse, crear tareas, iniciar sesiones de estudio y consultar estadísticas.

---

## Objetivos de UX

- **Simplicidad:** Interfaz limpia que no distraiga al usuario durante sus sesiones de estudio.
- **Claridad en la gestión del tiempo:** Reducir la fricción al momento de crear tareas, asignar prioridades y registrar sesiones.
- **Accesibilidad:** Diseño legible y navegable para cualquier tipo de usuario, priorizando estudiantes.
- **Confianza:** Visualización clara del progreso para motivar al usuario a mantener sus hábitos de estudio.
